What & Why
Field agents doing high-volume belt/gate scanning need a dedicated, distraction-free interface that shows an immediate color-coded full-screen result for every scan — no forms, no menus, just the signal, bag tag, pilgrim name, and accommodation. The existing Scan page is too complex for rapid belt scanning where agents process hundreds of bags in minutes.

Done looks like
A new "Rapid Scan" page accessible from the sidebar for admin, duty_manager, and airport_ops roles only
Full-screen color flash for 1.5 seconds after each scan with three states:
🟢 Green — valid Hajj bag with accommodation assigned: shows hotel name and address
🟡 Amber — valid Hajj bag but no accommodation assigned: shows "Manifested – Not Assigned"
🔴 Red — unrecognized tag, non-Hajj bag, or no Nusuk match: shows the reason
The last scanned bag tag and (if matched) the pilgrim's name are shown on screen
A session counter tracks green, amber, and red scan counts for the current shift
Flight context selector at the top so the agent can optionally associate scans to a flight
Hardware barcode scanner (keyboard wedge) works without clicking — input auto-focuses
Camera scan button available as a fallback
Offline-capable: uses the same offline queue as the existing Scan page; green/amber scans are queued with event type COLLECTED_FROM_BELT when offline
Red scans are logged to the red_scan_events table (same logic as existing Scan page)
Audio feedback: distinct tones via Web Audio API (no external files):
Green: pleasant two-note chime
Amber: single mid-tone beep
Red: low buzz
Arabic/English bilingual with proper RTL layout
Session can be reset with a "Clear session" button
Out of scope
Access for logistics_supervisor, ops_manager, driver, or other roles
Event type selection (fixed to COLLECTED_FROM_BELT)
Signature capture or photo upload
Delivery / handover workflow
Group-level bulk receive
